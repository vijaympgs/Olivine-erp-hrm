## **2. PURPOSE**

### **2.1 Business Purpose**
The Task Management module serves as the comprehensive transaction system for creating, assigning, tracking, and managing tasks across the organization. It provides teams with powerful tools for task creation, workflow automation, deadline management, and performance tracking. The module enables systematic task management with automated assignment, priority-based routing, progress monitoring, and integration with CRM records to ensure efficient task execution, improved team productivity, and enhanced project visibility through streamlined task processes and automated workflows.

### **2.2 Business Scope**

**In Scope:**
- Comprehensive task creation with templates, recurring tasks, and automated task generation
- Task assignment with intelligent routing, workload balancing, and skill-based assignment
- Task tracking with progress monitoring, status updates, and milestone management
- Workflow automation with approval processes, escalation triggers, and notification systems
- Integration with CRM records for contextual task information and customer relationship management
- Team collaboration with task comments, file attachments, and real-time updates
- Performance analytics with productivity metrics, completion rates, and team workload analysis
- Mobile-responsive interface for on-the-go task management and status updates
- Integration with calendar systems for deadline management and time tracking
- Automated reporting with task completion analytics and team performance insights

**Out of Scope:**
- Advanced project management and Gantt chart functionality (handled by external project management tools)
- Resource planning and capacity management (handled by dedicated resource planning modules)
- Advanced time tracking and billing systems (handled by external time tracking systems)
- Advanced document management and file storage (handled by external document systems)
- Advanced analytics and reporting (handled by dedicated reporting modules)

### **2.3 Key Stakeholders**
- **Sales Representatives:** Primary users for creating and managing sales-related tasks and follow-ups
- **Customer Service Agents:** Create and manage support tasks and customer service follow-ups
- **Team Leads:** Monitor team task performance, assign tasks, and manage team workload
- **Project Managers:** Oversee project tasks, manage deadlines, and coordinate team activities
- **Marketing Teams:** Create and manage campaign-related tasks and marketing activities
- **Executives:** Review overall task metrics, team productivity, and operational efficiency

- **Comment Integrity:** Comments cannot exist without a parent task
- **Hierarchy Integrity:** Sub-tasks cannot have different parent assignments than their parent task

- **Assignment Rules:** Assigned users must have appropriate permissions and availability
- **Hierarchy Rules:** Sub-tasks cannot be assigned to different users than parent task without approval
- **Completion Rules:** Tasks cannot be marked complete if sub-tasks are incomplete

### **6.2 Validation Logic**

```python
def clean_due_date_validation(self):
    """
    Validate task due date and time management
    """
    if self.start_date and self.due_date:
        if self.due_date <= self.start_date:
            raise ValidationError("Due date must be after start date")
    
    # Check parent task due date for sub-tasks
    if self.parent_task_id:
        try:
            parent_task = Task.objects.get(id=self.parent_task_id)
            if parent_task.due_date and self.due_date:
                if self.due_date > parent_task.due_date:
                    raise ValidationError("Sub-task due date cannot be after parent task due date")
        except Task.DoesNotExist:
            pass

def clean_completion_percentage(self):
    """
    Validate task completion percentage based on status
    """
    if self.status == 'completed' and self.completion_percentage != 100:
        self.completion_percentage = 100
    elif self.status == 'pending' and self.completion_percentage > 0:
        self.completion_percentage = 0
    elif self.status == 'in_progress' and self.completion_percentage == 0:
        self.completion_percentage = 10  # Minimum progress for in-progress

def clean_status_transitions(self):
    """
    Validate that status transitions follow workflow rules
    """
    if self.pk:  # Only validate on update
        try:
            old_task = Task.objects.get(pk=self.pk)
            if old_task.status != self.status:
                # Check if transition is allowed
                current_status = TaskStatus.objects.filter(status_code=old_task.status).first()
                if current_status and self.status not in current_status.next_statuses:
                    raise ValidationError(f"Cannot transition from {old_task.status} to {self.status}")
        except Task.DoesNotExist:
            pass

def clean_subtask_completion(self):
    """
    Validate that parent tasks cannot be completed if sub-tasks are incomplete
    """
    if self.status == 'completed' and self.pk:
        incomplete_subtasks = Task.objects.filter(
            parent_task_id=self.pk,
            status__in=['pending', 'in_progress'],
            is_deleted=False
        ).count()
        
        if incomplete_subtasks > 0:
            raise ValidationError("Cannot complete task with incomplete sub-tasks")
```

## **9. API SPECIFICATIONS**

### **9.1 REST Endpoints**

#### **GET /api/tasks/**
- **Purpose:** Retrieve paginated list of tasks with filtering and search capabilities
- **Authentication:** JWT token required with company_id validation
- **Parameters:** page, page_size, task_type, status, assigned_to_user_id, customer_id, date_range
- **Response:** Paginated task list with total count and filtering metadata

#### **POST /api/tasks/**
- **Purpose:** Create new task with validation and automatic processing
- **Authentication:** JWT token with create_task permission
- **Request Body:** Task data with required fields validation and assignment logic
- **Response:** Created task object with validation warnings and processing results

#### **GET /api/tasks/{id}/**
- **Purpose:** Retrieve detailed task information including comments and attachments
- **Authentication:** JWT token with read_task permission
- **Parameters:** task_id (UUID), include_comments, include_attachments, include_subtasks
- **Response:** Complete task object with related data and associations

#### **PUT /api/tasks/{id}/**
- **Purpose:** Update existing task information with change tracking and validation
- **Authentication:** JWT token with update_task permission
- **Request Body:** Updated task fields with validation and processing requirements
- **Response:** Updated task object with change summary and processing status

#### **POST /api/tasks/{id}/assign/**
- **Purpose:** Assign task to user with validation and notification
- **Authentication:** JWT token with assign_task permission
- **Request Body:** Assignment configuration with user details and notification preferences
- **Response:** Assignment confirmation with user details and notification status

#### **POST /api/tasks/{id}/comments/**
- **Purpose:** Add comment to task with validation and notification
- **Authentication:** JWT token with update_task permission
- **Request Body:** Comment data with content and notification preferences
- **Response:** Created comment object with validation results and notification status

#### **GET /api/task-templates/**
- **Purpose:** Retrieve list of task templates with filtering capabilities
- **Authentication:** JWT token with read_reference_data permission
- **Parameters:** is_active, task_type, category
- **Response:** Task template list with filtering metadata

#### **GET /api/task-statuses/**
- **Purpose:** Retrieve list of task statuses with filtering capabilities
- **Authentication:** JWT token with read_reference_data permission
- **Parameters:** is_active, status_type
- **Response:** Task status list with filtering metadata
