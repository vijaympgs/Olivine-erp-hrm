## **2. PURPOSE**

### **2.1 Business Purpose**
The Meeting Notes module serves as the comprehensive master data system for creating, managing, and standardizing meeting templates, formats, and structured note-taking processes. It provides organizations with powerful tools for meeting template creation, standardized agenda management, action item tracking, and knowledge preservation. The module enables systematic meeting documentation with template-based note creation, automated action item generation, participant management, and integration with CRM records to ensure consistent meeting practices, improved knowledge retention, and enhanced follow-up accountability through standardized meeting processes and structured information management.

### **2.2 Business Scope**

**In Scope:**
- Meeting template creation with standardized formats, agenda structures, and section templates
- Meeting type management with different templates for various meeting categories and purposes
- Participant management with role-based access, attendance tracking, and permission controls
- Agenda management with standardized sections, time allocation, and topic organization
- Action item tracking with assignment, due dates, and completion monitoring
- Meeting note creation with rich text editing, attachment support, and version control
- Knowledge base integration with searchable meeting archives and cross-referencing
- Template customization with company-specific branding and workflow configurations
- Mobile-responsive interface for on-the-go meeting participation and note-taking
- Integration with calendar systems for meeting scheduling and reminder management

**Out of Scope:**
- Real-time meeting scheduling and calendar management (handled by external calendar systems)
- Video conferencing and meeting platform integration (handled by external meeting tools)
- Advanced document management and file storage (handled by external document systems)
- Advanced analytics and reporting on meeting effectiveness (handled by dedicated reporting modules)
- Real-time collaboration during meetings (handled by external collaboration tools)

### **2.3 Key Stakeholders**
- **Meeting Organizers:** Primary users for creating meeting templates, managing agendas, and documenting meetings
- **Team Leads:** Use standardized meeting formats for team meetings and performance reviews
- **Project Managers:** Create project-specific meeting templates and track action items
- **Sales Representatives:** Document client meetings and track follow-up activities
- **Executives:** Review meeting archives, track strategic initiatives, and monitor action item completion
- **Compliance Officers:** Ensure meeting documentation meets regulatory requirements and audit standards

### **6.2 Validation Logic**

```python
def clean_template_structure(self):
    """
    Validate template structure and agenda configuration
    """
    if not self.agenda_structure or len(self.agenda_structure) == 0:
        raise ValidationError("Template must have at least one agenda section")
    
    # Validate agenda section structure
    for section in self.agenda_structure:
        if 'section_code' not in section:
            raise ValidationError("Each agenda section must have a section_code")
        if 'duration_minutes' not in section or section['duration_minutes'] <= 0:
            raise ValidationError("Each agenda section must have a valid duration")
    
    # Validate total time allocation
    total_time = sum(section.get('duration_minutes', 0) for section in self.agenda_structure)
    if self.default_duration_minutes and total_time > self.default_duration_minutes:
        raise ValidationError("Total agenda time cannot exceed default meeting duration")

def clean_time_allocation(self):
    """
    Validate time allocation and meeting duration
    """
    if self.agenda_time_allocation:
        total_allocated = sum(self.agenda_time_allocation.values())
        if self.default_duration_minutes and total_allocated > self.default_duration_minutes:
            raise ValidationError("Allocated time exceeds default meeting duration")

def clean_participant_roles(self):
    """
    Validate participant role configuration
    """
    all_roles = self.required_roles + self.optional_roles
    unique_roles = set(all_roles)
    
    if len(all_roles) != len(unique_roles):
        raise ValidationError("Participant roles must be unique across required and optional roles")
    
    # Validate against system roles
    valid_roles = get_system_role_choices()
    for role in all_roles:
        if role not in valid_roles:
            raise ValidationError(f"Invalid participant role: {role}")

def clean_template_uniqueness(self):
    """
    Validate template uniqueness within company
    """
    if MeetingTemplate.objects.filter(
        company_id=self.company_id,
        template_code=self.template_code
    ).exclude(pk=self.pk).exists():
        raise ValidationError("Template code must be unique within company")
```

## **9. API SPECIFICATIONS**

### **9.1 REST Endpoints**

#### **GET /api/meeting-templates/**
- **Purpose:** Retrieve paginated list of meeting templates with filtering and search capabilities
- **Authentication:** JWT token required with company_id validation
- **Parameters:** page, page_size, meeting_type, category, is_active, is_default
- **Response:** Paginated template list with total count and filtering metadata

#### **POST /api/meeting-templates/**
- **Purpose:** Create new meeting template with validation and structure verification
- **Authentication:** JWT token with create_meeting_template permission
- **Request Body:** Template data with required fields validation and structure verification
- **Response:** Created template object with validation warnings and processing results

#### **GET /api/meeting-templates/{id}/**
- **Purpose:** Retrieve detailed template information including structure and configuration
- **Authentication:** JWT token with read_meeting_template permission
- **Parameters:** template_id (UUID), include_sections, include_usage_stats
- **Response:** Complete template object with related data and usage statistics

#### **PUT /api/meeting-templates/{id}/**
- **Purpose:** Update existing template with version control and validation
- **Authentication:** JWT token with update_meeting_template permission
- **Request Body:** Updated template fields with validation and version control requirements
- **Response:** Updated template object with version information and change summary

#### **POST /api/meeting-templates/{id}/duplicate/**
- **Purpose:** Duplicate existing template with customization options
- **Authentication:** JWT token with create_meeting_template permission
- **Request Body:** Duplication configuration with new template details and customizations
- **Response:** Created template object with duplication confirmation and customization results

#### **GET /api/meeting-notes/**
- **Purpose:** Retrieve paginated list of meeting notes with filtering and search capabilities
- **Authentication:** JWT token with read_meeting_note permission
- **Parameters:** page, page_size, meeting_type, template_id, customer_id, date_range
- **Response:** Paginated meeting note list with total count and filtering metadata

#### **POST /api/meeting-notes/**
- **Purpose:** Create new meeting note from template with validation and processing
- **Authentication:** JWT token with create_meeting_note permission
- **Request Body:** Meeting note data with template reference and content
- **Response:** Created meeting note object with validation results and processing status

#### **GET /api/meeting-types/**
- **Purpose:** Retrieve list of meeting types with filtering capabilities
- **Authentication:** JWT token with read_reference_data permission
- **Parameters:** is_active, category
