### Step 1: Update Session State Tracker

Update `03_session_state_tracker.md` with current session's work:

1. **Update "Current Session Status"**
   - List all completed tasks
   - Document all deliverables created
   - Note any issues or blockers encountered

2. **Update "Last Session Summary"**
   - Move current session status to "Last Session Summary"
   - Document key findings
   - Note any decisions made

3. **Update "Next Session Roadmap"**
   - Plan next session priorities
   - Identify immediate actions
   - List pending tasks

4. **Update "Session Metrics"**
   - Tasks completed
   - Files modified
   - Files created
   - Lines of code written
   - Documentation updated
   - Issues resolved
   - Issues identified

5. **Update "Communication Log"**
   - Document recent communications
   - Note key decisions
   - Record feedback received

### Step 2: Verify Documentation

Check if any files were modified that need documentation:

1. **Check for modified files**
   - Review all files modified during session
   - Identify files that need documentation updates

2. **Update relevant markdown files**
   - Update documentation with changes made
   - Ensure code comments are clear where needed
   - Update any related guides or references

3. **Verify consistency**
   - Ensure all documentation is consistent
   - Check for outdated information
   - Update version numbers if applicable

### Step 3: Clean Up

Perform session cleanup:

1. **Close open files**
   - Close any open files that don't need to remain open
   - Verify no temporary or test files are left in production directories

2. **Check for errors**
   - Check for any console errors or warnings that need attention
   - Document any errors encountered
   - Note any workarounds applied

3. **Server status**
   - Note if servers are running or stopped
   - Document any server configuration changes
   - Record any database migrations that were applied

### Step 4: Quality Checks

Perform final quality checks:

1. **All deliverables created**
   - Verify all expected deliverables were created
   - Check file locations are correct
   - Ensure file names follow conventions

2. **All files in correct location**
   - Verify files are in correct directories
   - Check for any misplaced files
   - Ensure proper file structure

3. **All governance rules followed**
   - Verify no Location leakage in HRM code
   - Check UI consistency maintained
   - Ensure toolbar configuration correct
   - Verify copy-paste mergeability

4. **No Location leakage**
   - Search for any Location references in HRM code
   - Verify all imports use lazy string references
   - Check for any cross-module imports

5. **UI consistency maintained**
   - Verify all UIs follow UI canon
   - Check fonts, colors, spacing are correct
   - Ensure toolbar implementation is correct

### Step 5: Final Verification

Perform final verification:

1. **Read session state tracker**
   - Verify all updates are complete
   - Check for any missing information
   - Ensure formatting is correct

2. **Verify all sections complete**
   - Check all required sections are filled
   - Verify all checkboxes are checked
   - Ensure all metrics are updated

3. **Test functionality** (if applicable)
   - Test any code changes
   - Verify functionality works as expected
   - Check for any runtime errors

4. **Get user confirmation**
   - Present session summary to user
   - Confirm all tasks are complete
   - Get approval to end session
