## 2. PURPOSE

### 2.1 Business Purpose
The Organizational Chart module provides a comprehensive visual and structural representation of the company's hierarchy, reporting relationships, and organizational structure. It enables HR professionals, managers, and employees to understand the organizational design, reporting lines, and team structures at a glance.

### 2.2 Business Scope
**In Scope:**
- Hierarchical organization structure visualization
- Employee reporting relationships and direct/indirect reports
- Department and divisional structures
- Position management and organizational units
- Organizational change tracking and history
- Matrix organization support (dotted line reporting)
- Cost center and business unit associations
- Position vacancy visualization
- Succession planning integration

**Out of Scope:**
- Detailed employee personal information (handled in Employee Records)
- Compensation and salary data (handled in Compensation & Payroll)
- Performance management data (handled in Performance & Goals)
- Time and attendance tracking (handled in Time & Attendance)

### 2.3 Key Stakeholders
- **HR Professionals:** Structure management, reporting, compliance
- **Senior Management:** Strategic planning, organizational design
- **Department Managers:** Team structure, resource planning
- **Employees:** Understanding reporting relationships, organizational context
- **Auditors:** Compliance reporting, structure validation
- **IT Department:** System integration, data management

## 6. BUSINESS RULES & VALIDATIONS

### 6.1 Data Integrity Rules
- Organizational units cannot have circular parent-child relationships
- An organizational unit cannot be its own parent
- Manager must be an active employee in the same or parent unit
- Position headcount cannot be less than filled count
- Employee can only have one primary position assignment
- Reporting relationships must follow organizational hierarchy

### 6.2 Validation Logic
```python
def clean_parent_unit(self):
    """
    Validate parent unit to prevent circular references
    """
    if self.parent_unit_id and self.parent_unit_id == self.id:
        raise ValidationError("An organizational unit cannot be its own parent")
    
    # Check for circular references
    if self.parent_unit_id:
        parent = OrganizationalUnit.objects.get(id=self.parent_unit_id)
        if self.is_ancestor_of(parent):
            raise ValidationError("Circular reference detected in organizational hierarchy")

def clean_headcount(self):
    """
    Validate headcount consistency
    """
    if self.filled_count > self.headcount:
        raise ValidationError("Filled count cannot exceed total headcount")
    
    self.vacancy_count = self.headcount - self.filled_count

def clean_effective_dates(self):
    """
    Validate effective date logic
    """
    if self.end_date and self.effective_date > self.end_date:
        raise ValidationError("End date must be after effective date")
```

### 6.3 Business Constraints
- Maximum organizational depth: 10 levels
- Maximum direct reports per manager: 15 (configurable)
- Minimum organizational unit name length: 2 characters
- Position codes must be unique within company
- Organizational unit codes must be unique within company

## 9. API SPECIFICATIONS

### 9.1 REST Endpoints

#### **GET /api/organizational-units/**
- **Purpose:** Retrieve organizational units tree structure
- **Authentication:** JWT token required
- **Parameters:** 
  - `parent_id` (UUID): Filter by parent unit
  - `unit_type` (string): Filter by unit type
  - `is_active` (boolean): Filter by active status
  - `include_children` (boolean): Include child units
- **Response:** JSON array of organizational units with hierarchy

#### **POST /api/organizational-units/**
- **Purpose:** Create new organizational unit
- **Authentication:** JWT token with admin permissions
- **Request Body:** Organizational unit data
- **Response:** Created organizational unit object

#### **PUT /api/organizational-units/{id}/**
- **Purpose:** Update organizational unit
- **Authentication:** JWT token with edit permissions
- **Request Body:** Updated organizational unit data
- **Response:** Updated organizational unit object

#### **DELETE /api/organizational-units/{id}/**
- **Purpose:** Deactivate organizational unit
- **Authentication:** JWT token with admin permissions
- **Response:** Success confirmation

#### **GET /api/positions/**
- **Purpose:** Retrieve positions within organizational units
- **Authentication:** JWT token required
- **Parameters:**
  - `org_unit_id` (UUID): Filter by organizational unit
  - `is_active` (boolean): Filter by active status
  - `job_grade` (string): Filter by job grade
- **Response:** JSON array of positions

#### **GET /api/organizational-chart/**
- **Purpose:** Generate complete organizational chart data
- **Authentication:** JWT token required
- **Parameters:**
  - `view_type` (string): tree, box, list, matrix
  - `depth` (integer): Maximum hierarchy depth
  - `include_vacancies` (boolean): Include vacant positions
- **Response:** Formatted organizational chart data

### 9.2 Data Formats

- **Response Format:** JSON
- **Error Format:**
