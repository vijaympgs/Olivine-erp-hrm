## **2. PURPOSE**

### **2.1 Business Purpose**
The Asset Maintenance module provides comprehensive management of asset maintenance activities, enabling scheduled maintenance planning, work order processing, maintenance cost tracking, and asset performance monitoring to optimize asset lifecycle management, ensure regulatory compliance, and maximize asset value across the organization's fixed asset portfolio.

### **2.2 Business Scope**
**In Scope:**
- Asset maintenance scheduling and planning management
- Work order creation, assignment, and processing workflows
- Preventive maintenance program configuration and execution
- Corrective maintenance request handling and resolution
- Maintenance cost tracking and budget management
- Asset performance monitoring and failure analysis
- Maintenance history tracking and compliance reporting
- Integration with asset categories and depreciation systems

**Out of Scope:**
- Asset acquisition and disposal processing (handled by Asset modules)
- Asset depreciation calculation (handled by Depreciation Methods module)
- Asset inventory and physical verification (handled by Asset Register module)
- Asset reporting and analytics (handled by Asset Reports module)
- Asset insurance and warranty management (handled by separate modules)

### **2.3 Key Stakeholders**
- **Maintenance Managers:** Maintenance scheduling, work order assignment, and team coordination
- **Asset Managers:** Asset performance monitoring and maintenance budget oversight
- **Finance Controllers:** Maintenance cost tracking and budget compliance monitoring
- **Maintenance Technicians:** Work order execution and maintenance activity logging
- **External Auditors:** Maintenance compliance verification and cost audit trails

## **6. BUSINESS RULES & VALIDATIONS**

### **6.1 Data Integrity Rules**
- Maintenance numbers must be unique within each company
- Work order numbers must be unique within each company
- Completion dates cannot be before scheduled dates
- Actual costs cannot be negative
- Maintenance schedules must have valid frequency intervals
- Work orders cannot be completed before start time

### **6.2 Validation Logic**

```python
def clean_maintenance_number(self):
    """
    Validate maintenance number format and uniqueness
    """
    if not re.match(r'^[A-Z0-9]{2,50}$', self.maintenance_number):
        raise ValidationError("Maintenance number must be 2-50 alphanumeric characters")
    
    # Check uniqueness within company
    if AssetMaintenance.objects.filter(
        company_id=self.company_id,
        maintenance_number=self.maintenance_number
    ).exclude(id=self.id).exists():
        raise ValidationError("Maintenance number must be unique within company")

def clean_dates(self):
    """
    Validate date sequence and business rules
    """
    if self.completion_date and self.maintenance_date:
        if self.completion_date < self.maintenance_date:
            raise ValidationError("Completion date cannot be before maintenance date")
    
    if self.scheduled_date and self.maintenance_date:
        if self.scheduled_date > self.maintenance_date:
            raise ValidationError("Scheduled date cannot be after maintenance date")

def clean_costs(self):
    """
    Validate cost amounts and business rules
    """
    if self.estimated_cost and self.estimated_cost < 0:
        raise ValidationError("Estimated cost cannot be negative")
    
    if self.actual_cost and self.actual_cost < 0:
        raise ValidationError("Actual cost cannot be negative")
    
    if self.labor_cost and self.labor_cost < 0:
        raise ValidationError("Labor cost cannot be negative")
    
    if self.parts_cost and self.parts_cost < 0:
        raise ValidationError("Parts cost cannot be negative")

def clean_work_order_timing(self):
    """
    Validate work order timing and duration
    """
    if self.scheduled_start_time and self.scheduled_end_time:
        if self.scheduled_start_time >= self.scheduled_end_time:
            raise ValidationError("Scheduled end time must be after start time")
    
    if self.actual_start_time and self.actual_end_time:
        if self.actual_start_time >= self.actual_end_time:
            raise ValidationError("Actual end time must be after start time")

def clean_schedule_frequency(self):
    """
    Validate maintenance schedule frequency
    """
    if self.frequency_interval <= 0:
        raise ValidationError("Frequency interval must be positive")
    
    if self.start_date and self.next_due_date:
        if self.next_due_date <= self.start_date:
            raise ValidationError("Next due date must be after start date")
```

## **9.1 REST Endpoints**

#### **GET /api/asset-maintenance/**
- **Purpose:** Retrieve all maintenance records for a company
- **Authentication:** JWT token required
- **Parameters:** company_id, asset_id, maintenance_type, status, date_range
- **Response:** List of maintenance records with work orders and schedules

#### **POST /api/asset-maintenance/**
- **Purpose:** Create new maintenance record
- **Authentication:** JWT token required
- **Request Body:** Maintenance record configuration with work order details
- **Response:** Created maintenance record with generated maintenance number

#### **GET /api/asset-maintenance/{id}/**
- **Purpose:** Retrieve specific maintenance record with full details
- **Authentication:** JWT token required
- **Parameters:** Maintenance record UUID
- **Response:** Complete maintenance record with work orders and cost breakdown

#### **PUT /api/asset-maintenance/{id}/**
- **Purpose:** Update existing maintenance record
- **Authentication:** JWT token required
- **Request Body:** Updated maintenance record configuration
- **Response:** Updated maintenance record with change tracking

#### **POST /api/asset-maintenance/{id}/complete/**
- **Purpose:** Complete maintenance record and update asset status
- **Authentication:** JWT token required
- **Request Body:** Completion details with actual costs and work performed
- **Response:** Maintenance completion confirmation with asset status update

#### **GET /api/maintenance-work-orders/**
- **Purpose:** Retrieve work orders for technicians and managers
