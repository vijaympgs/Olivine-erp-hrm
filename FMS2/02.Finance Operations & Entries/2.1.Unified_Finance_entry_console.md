## **2. PURPOSE**

### **2.1 Business Purpose**
The Unified Finance Entry Console provides a single, intelligent interface for handling all financial transactions including journal entries, sales invoices, purchase invoices, credit notes, debit notes, payment receipts, and payment vouchers with dynamic form adaptation, smart validation, and integrated cost center and budget mapping for streamlined financial operations.

### **2.2 Business Scope**
**In Scope:**
- Unified transaction entry interface with smart form adaptation
- Dynamic field display based on voucher type selection
- Integrated cost center and budget mapping across all transaction types
- Smart validation engine with type-specific business rules
- Batch processing capabilities for high-volume transactions
- Real-time balance updates and budget impact analysis
- Multi-currency support with automatic FX calculations
- Document attachment and workflow integration

**Out of Scope:**
- Budget template creation (handled by Budget Templates module)
- Budget approval workflows (handled by Budget Approval Workflow module)
- Financial reporting and analytics (handled by Analytics module)
- Bank reconciliation (handled by dedicated Bank Reconciliation module)

### **2.3 Key Stakeholders**
- **Accountants:** Daily transaction entry and validation
- **Finance Managers:** Transaction oversight and approval
- **Department Heads:** Cost center allocation and budget monitoring
- **Auditors:** Transaction audit trail and compliance verification
- **System Administrators:** Console configuration and user management

class UnifiedTransaction(models.Model):
    """
    Master unified transaction model handling all voucher types with dynamic adaptation

    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)

    created_by_user_id = models.UUIDField()

    voucher_type = models.CharField(
        max_length=30,
        choices=[
            ('JOURNAL_ENTRY', 'Journal Entry'),
            ('SALES_INVOICE', 'Sales Invoice'),
            ('PURCHASE_INVOICE', 'Purchase Invoice'),
            ('CREDIT_NOTE', 'Credit Note'),
            ('DEBIT_NOTE', 'Debit Note'),
            ('PAYMENT_RECEIPT', 'Payment Receipt'),
            ('PAYMENT_VOUCHER', 'Payment Voucher'),
        ]
    )
    transaction_date = models.DateField()
    reference_number = models.CharField(max_length=100, blank=True)
    description = models.TextField()
    
    # Amount Fields
    currency_id = models.UUIDField()
    exchange_rate = models.DecimalField(max_digits=10, decimal_places=6, default=1.0)
    total_amount = models.DecimalField(max_digits=20, decimal_places=2)
    total_tax = models.DecimalField(max_digits=20, decimal_places=2, default=0)
    
    # Status and Workflow
    status = models.CharField(
        max_length=20,
        choices=[
            ('DRAFT', 'Draft'),
            ('PENDING_APPROVAL', 'Pending Approval'),

## **6. BUSINESS RULES & VALIDATIONS**

### **6.1 Data Integrity Rules**
- Transaction must balance (debits = credits) for journal entries
- Voucher type-specific mandatory fields must be completed
- Cost center allocation required for expense transactions above threshold
- Budget validation prevents overspending when budget tracking enabled
- Multi-currency transactions must use valid exchange rates
- Transaction dates cannot be in closed accounting periods

def clean_transaction_balance(self):
    """Ensure transaction balances for accounting integrity"""
    total_debits = sum(line.debit_amount for line in self.lines.all())
    total_credits = sum(line.credit_amount for line in self.lines.all())
    if total_debits != total_credits:
        raise ValidationError("Transaction must balance (debits must equal credits)")

def validate_budget_availability(self):
    """Check budget availability for expense transactions"""
    if self.voucher_type in ['PURCHASE_INVOICE', 'PAYMENT_VOUCHER']:
        for line in self.lines.all():
            if line.budget_id and line.cost_center_id:
                budget = Budget.objects.get(id=line.budget_id)
                if budget.remaining_amount < line.debit_amount:
                    raise ValidationError(f"Insufficient budget for {line.description}")

## **9. API SPECIFICATIONS**

### **9.1 REST Endpoints**

#### **GET /api/unified-transactions/**
- **Purpose:** Retrieve list of unified transactions with filtering and pagination
- **Authentication:** JWT token required
- **Parameters:** voucher_type, status, date_from, date_to, company_id
- **Response:** Paginated list of transactions with nested lines

#### **POST /api/unified-transactions/**
- **Purpose:** Create new unified transaction with dynamic validation
- **Authentication:** JWT token required
- **Request Body:** Transaction header and lines with voucher-specific fields
- **Response:** Created transaction with generated transaction number

#### **GET /api/unified-transactions/{id}/**
- **Purpose:** Retrieve specific transaction with full details and relationships
- **Authentication:** JWT token required
- **Parameters:** Transaction UUID
- **Response:** Complete transaction object with lines and allocations

#### **PUT /api/unified-transactions/{id}/**
- **Purpose:** Update existing transaction (draft status only)
- **Authentication:** JWT token required
- **Request Body:** Updated transaction fields and lines
- **Response:** Updated transaction with validation results

#### **POST /api/unified-transactions/{id}/submit-for-approval/**
- **Purpose:** Submit transaction for approval workflow
- **Authentication:** JWT token required
- **Parameters:** Transaction UUID
- **Response:** Transaction status updated to PENDING_APPROVAL

#### **POST /api/unified-transactions/{id}/approve/**
- **Purpose:** Approve transaction for posting
- **Authentication:** JWT token with approval permissions required
- **Parameters:** Transaction UUID
- **Response:** Transaction status updated to APPROVED

#### **POST /api/unified-transactions/{id}/post/**
- **Purpose:** Post transaction to General Ledger
- **Authentication:** JWT token with posting permissions required
- **Parameters:** Transaction UUID
- **Response:** Transaction posted with GL entries created

#### **GET /api/voucher-type-configurations/**
- **Purpose:** Retrieve voucher type configurations for dynamic form rendering
- **Authentication:** JWT token required
- **Parameters:** voucher_type, company_id
- **Response:** Voucher configuration with field definitions and validation rules
