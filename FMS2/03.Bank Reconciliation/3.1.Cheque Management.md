# 2. PURPOSE

## 2.1 Business Purpose
Cheque Management module enables organizations to manage the complete lifecycle of cheque payments, from issuance to clearance and reconciliation. This module provides comprehensive control over cheque transactions, prevents fraud, ensures proper authorization, and maintains accurate records for audit and compliance purposes.

## 2.2 Business Scope
- Cheque book management and inventory tracking
- Cheque issuance and printing with security features
- Multi-level approval workflows for cheque payments
- Cheque status tracking (issued, presented, cleared, bounced, stopped)
- Post-dated cheque (PDC) management and monitoring
- Cheque bounce handling and reissuance processes
- Bank reconciliation integration for cheque clearing
- Cheque stop payment and cancellation management
- Audit trail and compliance reporting

## 2.3 Key Stakeholders
- **Finance Teams:** Manage cheque issuance and approval workflows
- **Accountants:** Process cheque payments and reconciliations
- **Managers:** Approve cheque payments within authorization limits
- **Bank Reconciliation Teams:** Match cheque clearances with bank statements
- **Auditors:** Review cheque transactions and compliance controls
- **Management:** Monitor cheque payment workflows and cash flow

# 6. BUSINESS RULES & VALIDATIONS

## 6.1 Data Integrity Rules
- Cheque numbers must be unique within company and bank account
- Cheque amount must be positive and within approval limits
- Post-dated cheques cannot be presented before valid date
- Cheque books cannot exceed defined cheque ranges
- Stop payment requests require proper authorization

## 6.2 Validation Logic
```python
def clean_cheque_number(self):
    """
    Validate cheque number format and uniqueness
    """
    if not re.match(r'^\d{6,10}$', self.cheque_number):
        raise ValidationError("Cheque number must be numeric")
    
    if self.cheque_book_id:
        cheque_book = ChequeBook.objects.get(id=self.cheque_book_id)
        if not (cheque_book.book_start_number <= int(self.cheque_number) <= cheque_book.book_end_number):
            raise ValidationError("Cheque number not within book range")

def clean_amount(self):
    """
    Validate cheque amount
    """
    if self.amount <= 0:
        raise ValidationError("Cheque amount must be greater than zero")
    
    if self.amount > 99999999.99:
        raise ValidationError("Cheque amount exceeds maximum limit")

def clean_post_dated_cheque(self):
    """
    Validate post-dated cheque constraints
    """
    if self.is_post_dated and self.valid_from_date and self.valid_from_date <= self.cheque_date:
        raise ValidationError("Valid from date must be after cheque date for post-dated cheques")
```

## 6.3 Business Constraints

- Post-dated cheques cannot be presented before valid date
- Stop payment requests must be confirmed by bank
- Cheque books must be properly authorized and tracked

# 9. API SPECIFICATIONS

## 9.1 REST Endpoints

#### **GET /api/cheque-books/**
- **Purpose:** Retrieve all cheque books for the company
- **Authentication:** JWT token required
- **Parameters:** company_id, bank_account_id, status
- **Response:** List of cheque book objects with usage statistics

#### **POST /api/cheque-books/**
- **Purpose:** Create new cheque book
- **Authentication:** JWT token required
- **Request Body:** Cheque book details with number ranges
- **Response:** Created cheque book object with ID

#### **GET /api/cheques/**
- **Purpose:** Retrieve all cheques with filtering options
- **Authentication:** JWT token required
- **Parameters:** company_id, status, date_range, cheque_book_id
- **Response:** List of cheque objects with current status

#### **POST /api/cheques/**
- **Purpose:** Issue new cheque
- **Authentication:** JWT token required
- **Request Body:** Cheque details with payee and amount information
- **Response:** Created cheque object with approval workflow initiated

#### **POST /api/cheques/{id}/approve/**
- **Purpose:** Approve cheque payment
- **Authentication:** JWT token required
- **Request Body:** Approval details and comments
- **Response:** Updated cheque status and next approval level

#### **POST /api/cheques/{id}/stop-payment/**
- **Purpose:** Initiate stop payment request
- **Authentication:** JWT token required
- **Request Body:** Stop payment reason and details
- **Response:** Stop payment request object with tracking ID

#### **POST /api/cheques/{id}/print/**
- **Purpose:** Print cheque with security features
- **Authentication:** JWT token required
- **Request Body:** Printing template and format options
- **Response:** Print job status and tracking information

## 9.2 Data Formats
- **Request Format:** JSON with proper decimal precision for financial amounts
- **Response Format:** JSON with standardized cheque and approval data
- **Error Format:** JSON with error code, message, and validation details
