## **2. PURPOSE**

### **2.1 Business Purpose**
The Cost Centers module provides a framework for defining, managing, and tracking organizational cost centers for expense allocation, budget control, and financial analysis, enabling accurate cost accounting and responsibility reporting following Tally, NetSuite, and Dynamics 365 finance principles.

### **2.2 Business Scope**
**In Scope:**
- Cost center hierarchy and structure management
- Department and division cost center setup
- Project-based cost center configuration
- Cost center budget allocation and tracking
- Expense allocation and distribution rules
- Cost center reporting and analytics

**Out of Scope:**
- Transaction posting (handled by Journal Entries module)
- Budget creation (handled by Budget Management module)
- Employee management (handled by HR module)

### **2.3 Key Stakeholders**
- **Chief Financial Officer:** Strategic cost center structure oversight
- **Finance Managers:** Cost center budget allocation and monitoring
- **Department Heads:** Cost center expense management and reporting
- **Project Managers:** Project cost center tracking and control
- **Accountants:** Cost center transaction posting and reconciliation

## **6. BUSINESS RULES & VALIDATIONS**

### **6.1 Data Integrity Rules**
- Cost center codes must be unique within each company
- Parent cost centers must exist and be at higher level than child centers
- Budget amounts must be positive and within spending limits
- Allocation percentages must sum to 100% for each source
- Only one profit center can exist at each hierarchical level

### **6.2 Validation Logic**

```python
def clean_cost_center_code(self):
    """
    Validate cost center code format and uniqueness
    """
    if not re.match(r'^[A-Z0-9\-]+$', self.cost_center_code):
        raise ValidationError("Cost center code can only contain uppercase letters, numbers, and hyphens")
    
    # Check uniqueness within company
    if CostCenter.objects.filter(
        company_id=self.company_id,
        cost_center_code=self.cost_center_code
    ).exclude(id=self.id).exists():
        raise ValidationError("Cost center code must be unique within company")

def clean_hierarchy(self):
    """
    Validate parent-child relationship and hierarchy consistency
    """
    if self.parent_cost_center_id:
        parent = CostCenter.objects.get(id=self.parent_cost_center_id)
        if parent.level >= self.level:
            raise ValidationError("Parent cost center must be at higher level than child center")
        if parent.company_id != self.company_id:
            raise ValidationError("Parent cost center must belong to same company")

def clean_profit_center(self):
    """
    Validate profit center configuration
    """
    if self.is_profit_center and self.is_investment_center:
        raise ValidationError("Cost center cannot be both profit center and investment center")
    
    # Check if parent is profit center (child cannot be if parent is)
    if self.parent_cost_center_id:
        parent = CostCenter.objects.get(id=self.parent_cost_center_id)
        if parent.is_profit_center and self.is_profit_center:
            raise ValidationError("Child cost center cannot be profit center if parent is profit center")

def clean_spending_limit(self):
    """
    Validate spending limit configuration
    """
    if self.spending_limit and self.spending_limit < 0:
        raise ValidationError("Spending limit cannot be negative")
    
    # Check parent spending limit if child has higher limit
    if self.parent_cost_center_id and self.spending_limit:
        parent = CostCenter.objects.get(id=self.parent_cost_center_id)
        if parent.spending_limit and self.spending_limit > parent.spending_limit:
            raise ValidationError("Child spending limit cannot exceed parent limit")
```

### **6.3 Business Constraints**

- Cannot deactivate cost centers with open budgets
- Cannot modify hierarchy if allocations exist
- Cannot delete parent cost centers with active child centers
- Budget periods must be within fiscal year boundaries
