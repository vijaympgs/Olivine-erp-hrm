## **2. PURPOSE**

### **2.1 Business Purpose**
The Tax Codes & Rates module provides comprehensive configuration and management of standardized tax codes and their associated rates across different tax types and jurisdictions, enabling consistent tax calculation, compliance reporting, and financial integration within the organization's tax management framework.

### **2.2 Business Scope**
**In Scope:**
- Tax code definition and standardization
- Tax rate configuration and management
- Multi-jurisdiction tax rate support
- Tax type categorization and classification
- Effective date management for tax rates
- Tax code hierarchy and relationships
- Integration with financial transactions
- Tax reporting and compliance support

**Out of Scope:**
- Actual tax transaction processing (handled by transaction modules)
- Tax return filing (handled by Tax Returns module)
- Tax payment processing (handled by Payment modules)
- Tax audit and compliance reporting (handled by Tax Compliance module)

### **2.3 Key Stakeholders**
- **Tax Managers:** Tax code and rate configuration management
- **Finance Controllers:** Tax liability oversight and reporting
- **Compliance Officers:** Tax code standardization and regulatory compliance
- **System Administrators:** Technical tax system maintenance
- **External Auditors:** Tax configuration validation and audit support

## **6. BUSINESS RULES & VALIDATIONS**

### **6.1 Data Integrity Rules**
- Tax codes must be unique within each company
- Tax rates cannot overlap for the same tax code and effective date
- Tax code hierarchies cannot create circular dependencies
- Tax rate tiers must be sequential and non-overlapping
- System-defined tax codes cannot be deleted or modified

### **6.2 Validation Logic**

```python
def clean_tax_code(self):
    """
    Validate tax code format and uniqueness
    """
    if not re.match(r'^[A-Z0-9]{2,20}$', self.tax_code):
        raise ValidationError("Tax code must be 2-20 uppercase alphanumeric characters")
    
    # Check uniqueness within company
    if TaxCode.objects.filter(
        company_id=self.company_id,
        tax_code=self.tax_code
    ).exclude(id=self.id).exists():
        raise ValidationError("Tax code must be unique within company")

def clean_rate_percentage(self):
    """
    Validate tax rate percentage is within valid range
    """
    if self.rate_percentage < 0 or self.rate_percentage > 100:
        raise ValidationError("Tax rate percentage must be between 0 and 100")

def clean_tier_from_to(self):
    """
    Validate tier range is valid
    """
    if self.tier_from < 0:
        raise ValidationError("Tier from amount must be non-negative")
    
    if self.tier_to and self.tier_from >= self.tier_to:
        raise ValidationError("Tier from must be less than tier to")

def clean_tier_order(self):
    """
    Validate tier order is sequential
    """
    if self.tier_order < 0:
        raise ValidationError("Tier order must be non-negative")

def clean_effective_dates(self):
    """
    Validate effective and expiry dates
    """
    if self.effective_date and self.expiry_date:
        if self.effective_date >= self.expiry_date:
            raise ValidationError("Effective date must be before expiry date")

def clean_parent_tax_code(self):
    """
    Validate parent tax code to prevent circular dependencies
    """
    if self.parent_tax_code_id:
        # Check for circular reference
        current = self.parent_tax_code_id
        while current:
            if current == self.id:
                raise ValidationError("Circular tax code reference detected")
            try:
                parent = TaxCode.objects.get(id=current)
                current = parent.parent_tax_code_id
            except TaxCode.DoesNotExist:
                break
```

### **6.3 Business Constraints**
- Cannot delete tax codes with active tax rates
- Cannot modify tax rates if they are used in active transactions
- Cannot deactivate tax codes used in current fiscal period
- Tax rate tiers must be properly sequenced and non-overlapping
- System tax codes require special permissions for modification

## **9. API SPECIFICATIONS**

## **9.1 REST Endpoints**

#### **GET /api/tax-codes/**
- **Purpose:** Retrieve all tax codes for a company
- **Authentication:** JWT token required
- **Parameters:** company_id, jurisdiction_id, tax_type, is_active
- **Response:** List of tax code objects with hierarchical structure

#### **POST /api/tax-codes/**
- **Purpose:** Create new tax code
- **Authentication:** JWT token required
- **Request Body:** Tax code configuration with validation rules
- **Response:** Created tax code object with validation results

#### **PUT /api/tax-codes/{id}/**
- **Purpose:** Update existing tax code
- **Authentication:** JWT token required
- **Request Body:** Updated tax code configuration
- **Response:** Updated tax code object with change tracking

#### **GET /api/tax-rates/**
- **Purpose:** Retrieve tax rates for tax codes
- **Authentication:** JWT token required
- **Parameters:** company_id, tax_code_id, effective_date, is_active
- **Response:** List of tax rate objects with applicability dates

#### **POST /api/tax-rates/**
- **Purpose:** Create new tax rate
- **Authentication:** JWT token required
- **Request Body:** Tax rate configuration with tax code reference
- **Response:** Created tax rate object with validation results

#### **GET /api/tax-rate-tiers/**
- **Purpose:** Retrieve tax rate tiers for slab-based calculations
- **Authentication:** JWT token required
- **Parameters:** company_id, tax_rate_id, is_active
- **Response:** List of tax rate tier objects with range definitions

#### **POST /api/tax-calculate/**
- **Purpose:** Calculate tax for transaction
- **Authentication:** JWT token required
- **Request Body:** Transaction amount, tax code, date, entity profile
- **Response:** Tax calculation results with applied rates and tiers

## **9.2 Data Formats**
